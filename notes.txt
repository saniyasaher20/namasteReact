# PROS OF USING TAILWIND

- Consistency
- Reusability
- Maintanable
- Faster code
- Unused CSS removed

# CONS OF USING TAILWIND

- Initial learning curve

___________________________________________________

# TAILWIND

- CSS on the go (in the same file)
- Reusability
- Less bundle size
- Flexible UI = very much customizable

# INSTALLATION

   $ npm i -D tailwindcss  postcss
   $ npx tailwindcss init


# .postcssrc

Create a .postcssrc file in your project root, and enable the tailwindcss plugin.
    {
        "plugins": {
            "tailwindcss": {}
        }
    }

# HOW TO USE IT?

- Write this on top of index.css
    @tailwind base;
    @tailwind components;
    @tailwind utilities;


# SQUARE BRACKET NOTATION
     we don't have 200px predefined class in tailwind, so we will use square bracket notation.
     class="w-[200px]"
